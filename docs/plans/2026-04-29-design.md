## Overview

`wallet-self-audit` — локальная CLI-утилита для Mac M-series, которую владелец Bitcoin-кошелька запускает на **своих собственных** данных **до депозита крупных сумм**, чтобы проверить, не сгенерирован ли кошелёк уязвимым PRNG (Milk Sad CVE-2023-39910, Randstorm) и не содержат ли его исторические подписи коллизий nonce / смещения, позволяющего восстановить ключ.

**Главное архитектурное решение:** инструмент **детектирует** уязвимость и возвращает **вердикт** (`SAFE` / `SUSPICIOUS` / `VULNERABLE`) — но **не выводит восстановленный приватный ключ**. Если кошелёк скомпрометирован, единственная рекомендация: «срочно переведите средства в новый, безопасно сгенерированный кошелёк». Owner-mode authentication (BIP-322 signed message) гарантирует, что инструмент нельзя использовать против чужих адресов.

Этот контракт зашит в типах: возвращаемое значение всех аудит-функций — `VerdictWithoutKey`, frozen dataclass, который структурно неспособен нести 32-байтный секрет.

## Mission Statement & Anti-Scope

### Mission
Дать продвинутым Bitcoin-пользователям лучший в мире локальный инструмент превентивного криптоаудита **своего** кошелька на Mac M-series.

### Job Story (JTBD)
КОГДА я собираюсь положить значимые средства в Bitcoin-кошелёк, сгенерированный возможно небезопасным софтом,
→ я хочу запустить локальный аудит на известные PRNG/nonce уязвимости, не передавая никому свои секреты,
→ ЧТОБЫ убедиться что мой ключ не уязвим, и переложить средства если он уязвим — до того как меня дренируют.

### Что мы делаем (in-scope v1.0)
- ✅ Owner-mode проверка mnemonic / seed / xpub на Milk Sad (mt19937 + 32-bit timestamp)
- ✅ Owner-mode проверка mnemonic / seed на Randstorm (V8 Math.random ≤2³² энтропии, dominant variant)
- ✅ Owner-mode проверка mnemonic на Brainwallet (SHA-256 от популярных фраз)
- ✅ Nonce-audit для своего адреса (с BIP-322 proof-of-ownership): r-collision, lattice/HNP bias detection
- ✅ Только Bitcoin (P2PK, P2PKH, P2WPKH, P2WSH)
- ✅ Mac M-series native (coincurve ARM64, multiprocessing на P-cores)
- ✅ Local-only, air-gap friendly, mnemonic не уходит на диск никогда

### Что мы НЕ делаем (anti-scope)
- ❌ Не восстанавливаем приватные ключи (ни в логах, ни в отчётах, ни в стдауте)
- ❌ Не работаем с чужими адресами (gating через BIP-322)
- ❌ Не поддерживаем Taproot (Schnorr — другая математика)
- ❌ Не поддерживаем альткоины
- ❌ Не работаем с зашифрованными wallet.dat / keystore.json (это v2 — Hashcat path)
- ❌ Не сканируем блокчейн в фоне, не запускаемся как daemon
- ❌ Не пишем GPU-кернелы (mlx) в v1.0 — CPU достаточно для self-audit нагрузки
- ❌ Не используем Claude как backend для крипто-математики (только для генерации отчёта)

## Detailed Analysis

### 1. Owner-Mode Authentication

**Эксперт:** Troy Hunt
**Решение:** BIP-322 signed message gate (HARD) для `nonce_audit` + offline-only enforcement для `prng_audit`.

**Контракт:**
```python
class OwnershipProof(Protocol):
    method: Literal["bip322", "self_spend", "informational"]
    address: str
    challenge: str            # tool-generated, UTC-day-bound
    proof: bytes              # bip322 sig OR confirmed txid
    def verify(self) -> bool: ...   # pure-local; raises on tampering
```

**Алгоритм для nonce_audit:**
1. Tool генерирует challenge: `wallet-self-audit::<address>::<utc-date>::<random-nonce>`
2. User подписывает challenge через свой кошелёк (Sparrow / Electrum / Bitcoin Core 25+ / Trezor / Ledger fw≥2.x — все поддерживают BIP-322)
3. Tool верифицирует подпись локально через libsecp256k1
4. Challenge привязан к UTC-дате (24h окно) → защита от replay-атак
5. Без верификации → `OwnershipRequired` exception, fetch блокчейн-данных не запускается

**Алгоритм для prng_audit:**
- prng_audit принимает mnemonic/seed/xpub через `getpass` (interactive only, NO `--mnemonic` CLI flag — shell history leak).
- Владение seed-фразой = владение ключом → дополнительный proof не нужен.
- НО: tool **отказывается работать при включённом сетевом интерфейсе** (default), требует `--allow-online` override + airgap warning.

**Informational mode** (`--informational` flag): выводит только `collisions_found: bool` + `outgoing_tx_count`. Без `r`, `s`, txid, pubkey. Это публичная информация, риска нет.

**Отвергнутые варианты:**
- Self-spend gate (sub-dust transaction) — оставлен как fallback для legacy кошельков без BIP-322 поддержки. Минусы: on-chain footprint + ~$1 fee.
- xpub-only audit для nonce_audit — отвергнут (xpub доказывает наблюдение, не контроль).

**Риски:**
1. BIP-322 имплементации различаются между кошельками (Sparrow vs Electrum vs Core). Mitigation: ship cross-wallet test vectors per supported wallet.
2. Ledger firmware <2.x не поддерживает BIP-322 → fallback на self-spend.
3. UTC-date boundary рвётся при аудите через полночь UTC → use 24h sliding window.

### 2. No-Key-Leak Architecture

**Эксперт:** Troy Hunt + Filippo Valsorda
**Решение:** Hybrid:
- **B.2 для r-collision:** verify-by-pubkey-projection — ключ никогда не материализуется в памяти.
- **B.1 для lattice/HNP:** hash-and-burn — `d` существует <1ms, сразу `sha256(d || domain_sep)[:16]` → fingerprint, потом `zero_memory(d)`.

**Главный контракт всей системы:**
```python
@dataclass(frozen=True)
class VerdictWithoutKey:
    address: str
    status: Literal["SAFE", "SUSPICIOUS", "VULNERABLE"]
    finding: Literal["none", "r_collision", "lattice_bias", "weak_prng_milksad",
                     "weak_prng_randstorm", "brainwallet"]
    confidence: float                  # 0.0–1.0
    key_fingerprint: Optional[str]     # sha256(d || domain_sep)[:16], NEVER d itself
    recommendation: str                # "Move funds to a fresh wallet NOW."
    evidence_refs: List[str]           # txids only — no r, s, z, pubkeys
    audit_id: str
    # Class invariant enforced in __post_init__:
    #   no field may contain >16 hex chars besides audit_id and txids.
```

**Hardening process** (`with hardened_process():` контекст-менеджер):
1. `mlock()` всей памяти процесса — запрет swap. macOS: `mlockall` через `ctypes`.
2. Disable core dumps: `resource.setrlimit(RLIMIT_CORE, (0,0))`.
3. Отказ работать в Jupyter/IPython (caches `_`, `__`, `___`).
4. Все приватные данные — в `bytearray`, overwrite на `0x00 * 32` после использования. Python int нельзя занулить.
5. CI grep test: ищет fingerprint известного тест-ключа в stdout/stderr/logs во всех тестах. Build fail если утечка.

**verify-by-pubkey-projection** (для r-collision):
- Не вычисляем `d = (s_a*z_b - s_b*z_a) / ((s_b - s_a) * r) mod n` напрямую.
- Вместо этого: вычисляем `Q_recovered` через групповые операции на эллиптической кривой, сравниваем с `Q_known` из блокчейна.
- Если совпало — выводим `VULNERABLE` с fingerprint `sha256(formula_inputs || domain_sep)[:16]`.
- `d` никогда не существует как 256-битное число в памяти.

**Риски:**
1. B.2 математика тонкая — нужен тест-вектор с известно-уязвимым публичным адресом (Android Wallet 2013 case `9ec4bc...`) для валидации.
2. `mlock` требует rlimit-привилегий; degrade gracefully с warning, не fail-closed.
3. Python GC и string interning могут оставлять копии секретов. Real fix — Rust extension в v2; v1.0 acceptable: bytearrays + mlock + no core dumps + explicit zero.

### 3. PRNG Audit Module

**Эксперт:** Raymond Hettinger
**Решение:** Pure Python mt19937 reproduction + **сравнение entropy напрямую** (skip PBKDF2 — главное ускорение) + multiprocessing на P-cores.

**Структура модуля:**
```
src/wallet_self_audit/prng/
├── __init__.py
├── owner_input.py        # getpass + xpub mode, never disk
├── milk_sad.py           # mt19937 timestamp brute force
├── randstorm.py          # V8 Math.random MWC variant
├── brainwallet.py        # SHA-256 of common phrases
├── mt19937_cpp.py        # C++-compatible reproduction
├── bip39.py              # entropy ↔ mnemonic ↔ seed (no PBKDF2 needed)
├── derive.py             # BIP-32/44/49/84 (coincurve)
├── workers.py            # multiprocessing.Pool, stoppable, progress
└── fixtures/
    ├── milksad_truth.json     # disclosed addresses + timestamps
    ├── mt19937_cpp_ref.json   # 50 (seed → first 8 outputs) pairs
    └── brainwallet_seed.txt   # ~1.2M phrases (mmap target)
```

**Milk Sad алгоритм:**
1. User вводит mnemonic через getpass → конвертируем в 16-байт entropy (БИП-39 reverse via wordlist индексы).
2. Window: default `2017-01-01 → 2023-08-15` (CVE patch date), narrow if user gives `--created-after / --created-before`.
3. Per timestamp t: `state = init_genrand(t)` (custom 6-line implementation, валидируется fixture'ами) → 624 uint32s → `setstate()` в `random.Random` → 4× `genrand_int32` big-endian pack → 16 bytes entropy.
4. **Сравниваем entropy байты напрямую** — не идём через BIP-39 → seed → BIP-32. Это даёт ~5000× ускорение.
5. Stop on first match.

**Randstorm алгоритм:**
- Window: 2³² seeds, dominant V8 MWC variant 2011-2014.
- `--randstorm-fast` = top 2²⁸ "live" seeds → ~3 минуты на M3 Pro 8 cores.
- Документировано: покрывает только dominant variant. Output `SUSPICIOUS — partial coverage` если wallet vintage matches окно но не нашли.

**Brainwallet алгоритм:**
- Wordlist: RockYou top 1M + ~200K Bitcoin-themed (mmap).
- Per phrase: `sha256(phrase)` → privkey → derive P2PKH + P2WPKH addresses → match user's address set.
- Embarrassingly parallel, ~2M phrases/sec на 8 cores.

**Performance estimates на M3 Pro (6 P-cores, 18GB):**

| Module | Candidates | Per-candidate | 8-process | Notes |
|---|---|---|---|---|
| Milk Sad full window | 2.1×10⁸ | ~1.6µs | **~50 sec** | numpy-vectorized init_genrand |
| Milk Sad narrow (1yr) | ~3×10⁷ | ~1.6µs | **~7 sec** | typical case |
| Randstorm full | 4.3×10⁹ | ~5µs | **~45 min** | MWC variant only |
| Randstorm fast | 2.7×10⁸ | ~5µs | **~3 min** | |
| Brainwallet 1.2M | 1.2×10⁶ | ~4µs | **<1 sec** | coincurve dominates |

**mt19937 reproduction validation strategy** (CRITICAL):
1. Implement `init_genrand(seed)` from libc++/MSVC reference (1812433253 multiply, XOR with prev>>30). 6 lines.
2. **Fixture 1:** vendor 50 (seed, first 8 outputs) tuples generated by tiny C++ program in `tools/ref.cpp`. Test asserts byte-identical output. Required CI gate.
3. **Fixture 2:** vendor 3-5 disclosed Milk Sad mnemonics + their known timestamps (public, funds drained). Test: feed mnemonic + wide window → assert published timestamp recovered. **Module не shippable если этот тест не проходит.**
4. **Fixture 3:** entropy → mnemonic → entropy must be identity for 1000 random 128-bit values.
5. `_VALIDATED = check_fixtures()` at module load — refuses brute force if fixtures fail.

**Риски:**
1. **False negatives feel like SAFE.** Output должен говорить `SUSPICIOUS — not vulnerable to checks performed` с явным списком проверенных векторов. Никогда не unqualified `SAFE`.
2. **mt19937 drift across stdlibs.** libc++ vs libstdc++ vs MSVC identical per std, но `bx seed` 3.x point releases могли использовать non-standard wrapper. Mitigation: vendor fixtures от ≥2 версий bx.
3. **getpass на скомпрометированном хосте — театр.** README must lead with "boot Tails USB, run airgapped, destroy USB". Documentation risk = P1.

### 4. Nonce Audit + Lattice (Verdict-Only)

**Эксперт:** Andrej Karpathy
**Решение:** Hybrid — pure Python LLL default, optional fpylll для BKZ; bias hypothesis enumeration в priority order; verdict-only output (никогда не выводим `d`).

**Pipeline:**
1. BIP-322 ownership проверена (см. Aspect 1).
2. Fetch all outgoing tx через mempool.space → extract `(r, s, pubkey)` pairs, compute `z` через btclib sighash.
3. **Stage A: r-collision check.** Group by pubkey, find duplicate r → `verify_recoverable_via_pubkey()` без вычисления `d`.
4. **Stage B: lattice/HNP check.** Только если Stage A clean.
   - Bias hypotheses в порядке приоритета:
     1. `top_bits_zero ∈ {1, 2, 4, 8}` (Boneh-Venkatesan construction)
     2. `low_bits_zero ∈ {1, 4, 8}` (Howgrave-Graham construction)
     3. `top_byte_constant` (histogram-based detection)
   - Required signature counts (hard threshold, refuse otherwise):
     - 1-bit bias → 220+ sigs
     - 4-bit bias → 70 sigs
     - 8-bit bias → 33 sigs
   - Per hypothesis: build B-matrix → run LLL → check shortest vector → candidate `d` → `pubkey_from(d) == known_pubkey`?
   - Hard timeout per hypothesis: 60s LLL, 5min BKZ.
   - Multiple hypotheses в multiprocessing.Pool параллельно.
5. **Output:** `VerdictWithoutKey` без `d`. Если recovery succeeded → fingerprint = `sha256(d || domain_sep)[:16]` → zero_memory(d).

**Library choice:**
- **Default:** vendored pure Python LLL (~200 LOC, Nguyen-Stehlé). Fine for dim ≤ 60. Uses `fractions.Fraction` for exact arithmetic.
- **Optional:** fpylll. Install via `brew install gmp mpfr fplll && pip install '.[lattice]'`. For dim >60 / BKZ.

**Anti-scope:**
- Polynonce attack (Kudelski 2023) — defer to v1.1. Stub `polynonce_audit.py` with TODO.

**Риски:**
1. Pure Python LLL numerical drift на dim>60 → mitigation `fractions.Fraction` (slow but exact).
2. fpylll install на M-series ещё фрагилен (gmp/mpfr) → optional, never required.
3. Wrong bias hypothesis тихо съедает minutes → hard timeout per hypothesis.

### 5. ECDSA Math Correctness (Sighash + Recovery Detection)

**Эксперт:** Martin Kleppmann
**Решение:** **btclib для sighash + coincurve для key math** + независимый pure-Python sighash для SIGHASH_ALL как cross-checker.

**Layered invariants:**
1. **Sighash preimage:** byte-for-byte identical to wallet's hash. Один wrong byte = garbage `z`.
2. **Hash:** `z = int.from_bytes(sha256(sha256(preimage)), 'big')`. Reduce mod n only inside recovery formula.
3. **Recovery:** modular arithmetic over secp256k1 n. Handle `s_a == s_b`, low-s normalization (BIP-62), `gcd(s_b-s_a, n) ≠ 1` (assert).
4. **Validation:** derive pubkey via coincurve, compare compressed AND uncompressed to blockchain pubkey. **No match = no claim.**

**v1.0 must-work features:**

| Feature | Required | Reason |
|---|---|---|
| Legacy SIGHASH_ALL (0x01) | YES | 99% of pre-2016 P2PKH txs |
| BIP-143 SIGHASH_ALL P2WPKH | YES | All segwit nonce-reuse cases since 2017 |
| Low-s normalization (BIP-62) | YES | Post-2015 sigs always low-s |
| Pubkey-match validation | YES | Hard gate before "VULNERABLE" |
| SIGHASH_NONE/SINGLE/ANYONECANPAY | NO (warn+skip) | Vanishingly rare, complex |
| OP_CODESEPARATOR | NO | Never in standard scripts |
| P2SH-wrapped segwit | NO (v1.1) | Different scriptCode derivation |

**Test vectors:**
1. BIP-143 official 3 canonical vectors → hardcode in `tests/test_sighash.py`.
2. Bitcoin Core `script_tests.json` → legacy sighash.
3. Real Android Wallet 2013 r-collision (txs `9ec4bc...` + `4a0a25c1...`) → end-to-end: fetch → compute z → verify recoverability → match.
4. Self-consistency: random testnet tx with known privkey, sign → extract → compute z → verify `s == k⁻¹(z + r·d) mod n`.

**Implementation order (one PR per layer):**
1. `sighash_builder.py` + canonical-vector tests (no recovery).
2. Real-tx integration test on johoe address (z must match published value).
3. `recovery_detector.py` (verify-by-pubkey-projection, не извлекает d).
4. End-to-end on johoe collision.

**Риски:**
1. btclib JSON adapter получает `prev_value` неправильно (sat vs BTC, byte order). Mitigation: assert against block-explorer amount.
2. coincurve `PrivateKey(d).public_key.format(compressed=True)` — match exact compression flag from blockchain. Try both.
3. Negative test: deliberate-corruption test that wrong z fails pubkey check.

### 6. Apple Silicon Native Optimizations

**Эксперт:** Hettinger + Karpathy
**Решение:** CPU-first stack: coincurve ARM64 + numpy/Accelerate + multiprocessing(P-cores). **Никаких GPU/mlx в v1.0.**

**Workload analysis:**

| Workload | Best optimization | Rationale |
|---|---|---|
| SHA-256 / SHA-512 | hashlib (ARM64 SHA2 instructions) + multiprocessing | Bitcoin ASICs существуют именно потому что GPU посредственны для SHA |
| HMAC-SHA512 / PBKDF2 | hashlib.pbkdf2_hmac (C) + multiprocessing | Sequential within password, parallel across N |
| secp256k1 mult | coincurve ARM64 wheel | Verified, just `pip install coincurve` |
| LLL/BKZ | numpy/Accelerate (default) для прототипа; fpylll опциональный | numpy на macOS arm64 auto-uses Accelerate (AMX coprocessor) |
| mt19937 | Pure CPython OK; Numba @njit если профиль покажет hot loop | Sequential, no SIMD |

**Stack для v1.0:**
```python
hashlib                  # stdlib, ARM64 SHA2 hardware
coincurve >= 19          # libsecp256k1, arm64 wheel
numpy >= 2.0             # Accelerate auto-link
multiprocessing          # stdlib — Pool(processes=6) targets P-cores
mnemonic >= 0.21         # BIP-39 wordlist + checksum
btclib >= 4              # sighash builder
fpylll  (optional)       # only for real BKZ; brew install fplll
numba   (optional)       # only after profiling
```

**P-core targeting:** `multiprocessing.Pool(processes=6)` на M3 Pro. macOS QoS schedules CPU-heavy workers на P-cores by default — не fight it через taskpolicy.

**Performance baseline на M3 Pro:**

| Operation | Throughput |
|---|---|
| SHA-256 single block (1 core) | ~600 MB/s |
| SHA-256 across 6 P-cores | ~3.5 GB/s aggregate (~55M short-input hashes/sec) |
| PBKDF2-HMAC-SHA512 2048 rounds (1 core) | ~250 ops/sec |
| PBKDF2 across 6 P-cores | **~1,500 BIP-39 seeds/sec** ← bottleneck |
| secp256k1 scalar mult (coincurve) | ~25k ops/sec |
| Full BIP-39 → BIP-32 → P2PKH | **~1,200/sec на 6 P-cores** |
| LLL dim 80 (numpy/Accelerate) | ~2-5 sec/instance |
| mt19937 advance (CPython) | ~50M/sec; Numba ~500M/sec |

**Defer to v1.1 — и почему:**
| Optimization | Why deferred |
|---|---|
| MLX/Metal SHA-256 kernel | No upstream lib; bit-rotation Metal-неудобный; 1-2 weeks Metal Shading Language work |
| UltrafastSecp256k1 Python binding | Existed C++20 with Metal backend; no Python wrapper; ~2 days pybind11 shim worth it когда secp станет bottleneck (а PBKDF2 dominates first) |
| Mass PBKDF2 on Metal | Real win (ND-range), but custom Metal kernel needed |

**Risks:**
1. fpylll install fragile на Apple Silicon (gmp/mpfr/fplll). Mitigation: pin install instructions, optional only.
2. coincurve wheel lag для новых Python версий. Mitigation: pin known-good `coincurve==N.N`.
3. **GPU temptation eats schedule.** Karpathy's law: профилирй py-spy сначала. Если PBKDF2 не >70% wall time, GPU portation = procrastination.

### 7. Plugin Architecture & Vector Registry

**Эксперт:** Martin Fowler
**Решение:** `@register_vector` decorator + 3-stage pipeline (Discovery → Analysis → Synthesis) + per-vector pydantic config schema + bulkhead failure isolation.

**Vector contract:**
```python
class Stage(Enum):
    DISCOVERY = 1     # fetch raw chain data (always runs once)
    ANALYSIS = 2      # independent vectors (parallelizable)
    SYNTHESIS = 3     # vectors consuming other vectors' outputs

class VectorPlugin(ABC):
    name: ClassVar[str]
    description: ClassVar[str]
    stage: ClassVar[Stage]
    requires: ClassVar[list[str]] = []
    config_schema: ClassVar[type[BaseModel]]

    @abstractmethod
    def applicability(self, ctx: AuditContext) -> bool: ...
    @abstractmethod
    def cost_estimate(self, ctx: AuditContext) -> Cost: ...
    @abstractmethod
    def run(self, ctx: AuditContext) -> VerdictWithoutKey: ...
```

**Shared context (lazy memoized fetchers):**
```python
class AuditContext:
    address: str
    config: dict
    cache: dict[str, Any]                # vector outputs, namespaced
    api_client: MempoolClient            # shared rate-limited

    def signatures(self) -> list[SignatureRecord]: ...   # memoized
    def transactions(self) -> list[Tx]: ...
    def address_metadata(self) -> AddressMeta: ...
    def put(self, key: str, value): ...   # namespaced storage
    def get(self, key: str, default=None): ...
```

**Bulkhead:** каждый vector в `try/except` shell. Crashing lattice → `VerdictResult(status=ERROR)`, nonce_audit results остаются.

**Auto-CLI flags:** `--<vector>.<knob>` (e.g. `--lattice.bias_bits=8`) генерируется из per-vector pydantic schema.

**Anti-scope:**
- Process isolation (subprocess per vector) — overkill в v1.
- Plugin marketplace, signed plugins — premature.
- Full DAG resolver — stages cover 95%.
- Hot-reload — CLI tool, not daemon.

### 8. Storage Layer (SQLite Cache)

**Эксперт:** Markus Winand
**Решение:** SQLite per-audit DB в `~/.local/share/wsa/audits/<audit_id>/cache.db`. BLOB(32) для r/s. Idempotent inserts. Hand-rolled migrations через `PRAGMA user_version`.

**Core schema:**
```sql
CREATE TABLE transactions (
    txid TEXT PRIMARY KEY,
    block_height INTEGER,
    raw_hex BLOB,
    fetched_at INTEGER NOT NULL
);

CREATE TABLE signatures (
    txid TEXT NOT NULL,
    vin_index INTEGER NOT NULL,
    pubkey_hex TEXT NOT NULL,
    r BLOB NOT NULL,        -- 32 bytes
    s BLOB NOT NULL,        -- 32 bytes
    z BLOB,
    sighash_type INTEGER NOT NULL,
    script_type TEXT NOT NULL,
    PRIMARY KEY (txid, vin_index),
    FOREIGN KEY (txid) REFERENCES transactions(txid)
) WITHOUT ROWID;

-- THE index — collision detection хот path
CREATE INDEX idx_sig_pubkey_r ON signatures(pubkey_hex, r);

CREATE TABLE address_pubkeys (
    address TEXT NOT NULL,
    pubkey_hex TEXT NOT NULL,
    first_seen_txid TEXT NOT NULL,
    PRIMARY KEY (address, pubkey_hex)
) WITHOUT ROWID;

CREATE TABLE audits (
    audit_id TEXT PRIMARY KEY,    -- uuid
    address TEXT NOT NULL,
    started_at INTEGER NOT NULL,
    finished_at INTEGER,
    toolkit_version TEXT NOT NULL,
    ownership_proof_hash TEXT NOT NULL,    -- BIP-322 sig hash
    status TEXT NOT NULL    -- 'running'|'safe'|'suspicious'|'vulnerable'|'error'
);

CREATE TABLE findings (
    finding_id INTEGER PRIMARY KEY,
    audit_id TEXT NOT NULL REFERENCES audits(audit_id),
    severity TEXT NOT NULL,
    kind TEXT NOT NULL,
    pubkey_hex TEXT,
    payload_json TEXT NOT NULL    -- redacted, no priv data
);
```

**Collision query (Winand):**
```sql
SELECT pubkey_hex, r, COUNT(*) AS c, GROUP_CONCAT(txid||':'||vin_index)
FROM signatures
WHERE pubkey_hex IN (SELECT pubkey_hex FROM address_pubkeys WHERE address = ?)
GROUP BY pubkey_hex, r HAVING c > 1;
```
Index-only scan, no table lookup.

**Encryption:** SQLCipher with key derived from BIP-322 proof hash. Лёгкий cost (~5-15%).

**GC:** 30-day auto-delete on tool startup if older than 24h. Critical findings pinned.

### 10. Logging & Redaction

**Эксперт:** Kelsey Hightower
**Решение:** stdlib `logging` + structlog adapter, hybrid allowlist+blocklist redaction filter, custom SECURITY=35 level, hash-chained audit log.

**Levels:**
| Level | Use |
|---|---|
| DEBUG | txid, r-prefix (first 6 hex), API latency |
| INFO | "vector=X address=bc1q...xyz stage=fetch_txs count=42" |
| WARNING | malformed DER, sighash stub, rate-limit backoff |
| **SECURITY (35)** | redaction triggered, ownership mismatch, recovery confirmed |
| ERROR | vector aborted, API down |

**Hybrid redaction filter:**
```python
SUSPECT_HEX = re.compile(r'\b[0-9a-fA-F]{64}\b')
SUSPECT_HEX_LONG = re.compile(r'\b[0-9a-fA-F]{128,}\b')
SUSPECT_BIP39 = re.compile(r'\b(?:[a-z]{3,8}\s+){11,23}[a-z]{3,8}\b')
BLOCKLIST_KEYS = {"privkey","d","mnemonic","seed","wif","xprv","s","k","nonce","sighash_z"}
ALLOWLIST_FIELDS = {"address","txid","vector","stage","count","level","ts","event","r_prefix"}
```

**Tamper-evident audit chain:** `~/.local/state/wsa/audit.jsonl`, append-only (`chflags uappend` macOS), hash-chained:
```
{"ts":"...","seq":N,"event":"key_recovered","address":"bc1q…","prev_hash":"...","hash":"..."}
```
Verifier CLI: `wsa audit verify`.

**Tests are mandatory:** `tests/test_redaction.py` feeds known privkeys/mnemonics → assert никогда не survive any code path. CI fails если fixture leaks.

### 11. Testing Strategy

**Эксперт:** Kent C. Dodds
**Решение:** Testing Trophy — unit (5-10) → **integration (10-15, highest ROI)** → property (5+) → E2E (1-2).

**Critical fixtures:**
1. **DER signatures** (`tests/fixtures/der_signatures.json`): low_s_normalized, high_r_with_leading_zero, sighash_anyonecanpay.
2. **BIP-143 canonical** (`tests/fixtures/bip143_vectors.json`): copy from BIP-143 spec.
3. **Synthetic vulnerable wallet** (`tests/fixtures/synthetic_reuse.py`): we control k, sign 2 messages, run pipeline, assert `VULNERABLE` verdict + correct fingerprint.
4. **mempool.space response** (`tests/fixtures/mempool/`): recorded JSON, mocked via `unittest.mock.patch`.
5. **Milk Sad ground truth** (`tests/fixtures/milksad_truth.json`): disclosed addresses + timestamps.

**Property tests (hypothesis):**
```python
@given(d=st.integers(1, SECP256K1_N - 1),
       z1=st.integers(1, SECP256K1_N - 1),
       z2=st.integers(1, SECP256K1_N - 1),
       k=st.integers(1, SECP256K1_N - 1))
def test_nonce_reuse_recovers_pubkey(d, z1, z2, k):
    assume(z1 != z2)
    r, s1 = sign(z1, d, k)
    _, s2 = sign(z2, d, k)
    Q_recovered = recover_pubkey_only(r, s1, s2, z1, z2)
    assert Q_recovered == pubkey_from(d)
```

**Security tests (non-negotiable):**
```python
def test_log_output_never_contains_private_key():
    d = 0xC0FFEE...
    captured = capture_stdout_stderr(lambda: run_audit(d))
    assert f"{d:064x}" not in captured
    assert str(d) not in captured
```

**Performance regression:**

- Adaptive thinking `effort=low` — это writing task, не reasoning.
- Prompt caching: ~90% off на 2-м wallet onwards.

**2. Pattern-naming pass (опционально)**
- After deterministic vectors complete: Claude получает summary (r-value octant histogram, k-bias score) и предлагает named hypotheses ("upper-octant clustering consistent with sign-bit bug").
- Claude — strictly **nominator**; deterministic verifier потом attempts named attack. LLM never gates positive finding.

**3. Claude Code plugin packaging (минимально)**
```
.claude/plugins/wallet-self-audit/
  plugin.json
  skills/wsa-audit/SKILL.md      # auto-invoke on "audit my wallet"
  mcp/wsa_server.py              # @tool wrappers
```
`@tool` decorator → in-process MCP, no IPC, secrets никогда не leave machine. **Tool result MUST be sanitized** через `summary_redacted()` before returning to Claude.

**Anti-patterns (HARD NO в v1.0):**
- Claude как orchestrator выбирающий vectors — нondeterminism убивает reproducibility forensic tool.
- Mnemonic / seed в Claude API calls — privacy leak.
- Claude как backend для crypto math — hallucination = wrong key.

**v1.1+ integrations:**
- Bounded subagent для ambiguous orchestration (только когда deterministic tree returns "inconclusive"). Hard task budget (Opus 4.7 task-budget feature) — 8 tool calls, 60s wall.
- Multi-wallet portfolio reports с cached system prompt.
- Hook on `PostToolUse(nonce_audit)` для pretty-print findings inline.
- Vision (3.75MP) для screenshot-of-paper-wallet OCR — v1.2.

**Risks:**
1. **Privacy leakage via tool results.** Tool results boundary = trust boundary. Enforce redaction layer в MCP server with unit test that greps tool outputs for hex≥64, base58≥51, BIP39 wordlist hits. Fail closed.
2. **LLM-blessed false positives.** Если `wsa explain` пишет "ваш кошелёк скомпрометирован" из-за плохого pattern-match, users panic-move funds. Mitigation: report template forces every claim cite `finding.code` from deterministic JSON.
3. **Reproducibility erosion.** Forensic tool must быть reproducible. Mitigation: persist `(model_id, prompt_hash, response, evidence_hash)` в audit record. Finding deterministic; prose versioned.
